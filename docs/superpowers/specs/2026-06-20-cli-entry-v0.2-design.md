# MiniCode CLI Entry v0.2 Design

## Goal

Turn MiniCode into an installable local CLI while preserving the existing
`python main.py` workflow and all v0.1.1 safety boundaries.

## Commands

```text
minicode "帮我分析这个项目"
minicode -w examples/sample_project "帮我分析这个项目"
minicode --debug "帮我分析这个项目"
minicode --mock "帮我分析这个项目"
minicode --version
minicode doctor
```

The legacy equivalents through `python main.py` remain supported.

## Structure

- `minicode_cli.py` owns argument parsing and command behavior.
- `main.py` is a compatibility wrapper that imports and calls
  `minicode_cli.main`.
- `pyproject.toml` exposes `minicode = "minicode_cli:main"`.
- Existing `agent`, `tools`, `safety`, and `memory` packages remain unchanged.

## Path behavior

`app_root` is the installed/source project directory containing
`skills/skills.json`. `workspace` is:

1. the resolved `--workspace` path when provided;
2. otherwise `Path.cwd().resolve()`.

Invalid or non-directory workspaces produce a clear parser-style error and a
non-zero exit code.

## Options

- positional `query`: zero or more words joined with spaces;
- `--workspace` / `-w`: target project directory;
- `--debug`: temporarily enables `MINICODE_DEBUG=1` for this process;
- `--mock`: temporarily removes model credentials for this process;
- `--version`: prints `MiniCode 0.2.0`;
- `doctor`: prints Python version, app root, workspace, skills configuration,
  memory path, and effective LLM mode.

`doctor` accepts `--workspace`. It never reads target project contents or writes
memory.

## Packaging

Use setuptools through `pyproject.toml`. Explicitly include:

- top-level `minicode_cli` and compatibility `main` modules;
- `agent`, `tools`, `safety`, and `memory` packages;
- `skills/skills.json` as installed data.

Editable installation is the supported workflow:

```text
python -m pip install -e .
```

Publishing to PyPI is outside this version.

## Testing

Add tests for argument parsing, workspace selection, debug/mock environment
behavior, version, doctor output, compatibility wrapper, editable install, and
installed console-script execution. Existing 48 tests and all old demo commands
must remain green.
